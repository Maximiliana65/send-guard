## 2026-07-16　設計スタート〜フェーズ1実装

### 話したこと・決めたこと
- そもそもの困りごと: AIとの会話でうっかりEnterや再試行ボタンを押して誤送信してしまう
- GitHubの既存拡張機能をそのまま入れるのは不安 → 自作する方針に
- Enterキーの挙動は「常に改行、送信は明示的な解除操作をした時だけ」に決定
- 送信ボタン・再試行ボタンの誤クリックも同じ仕組みでブロックすることに（Enterだけ対策しても根本解決にならないため）
- ロック解除は🔒アイコンのクリック＋ショートカット(Ctrl+Shift+U)の両方を用意
- 解除は「次の1回だけ」有効。送信したら自動でまた鍵がかかる（毎回慎重に、という方針）
- 対応サイトはまずClaude.aiのみ。動作確認後にChatGPT・Geminiへ拡大
- お楽しみ機能（一言コメント表示）は初期状態OFF。送信した瞬間に表示。コメントは`fun/comments.js`にただの配列として書くので、後から自分で気軽に追記できる形にした
- 保守性を重視し、「コア（共通ロジック）／サイトアダプター／お楽しみプラグイン」の3層構造に設計
- 将来的にGitHubで公開したいので、`_locales`を使った多言語対応の土台も最初から用意（今回は日本語＋英語）
- Gitはまずプライベートリポジトリで始める方針

### 作ったもの
- フェーズ1一式（`manifest.json`、`core/`、`adapters/claude.js`、`fun/`、`popup/`、アイコン、`_locales`）
- README / ROADMAP / CHANGELOG / この日誌
- 設計書（`docs/`）

### 次回への申し送り

- 動作確認しながら、バッジの表示位置やロック解除の感触を調整する可能性あり

## 2026-07-17　実際に使ってみてのフィードバック反映

### 実際に使ってみての感想(良かった点)
- ロック中の表示が控えめで邪魔にならない
- 「うっかり」がかなり予防できている実感あり
- 再試行ボタンのロックも好評

### 直した点
- 🔒アイコンをクリックのたびにロック⇔解除が切り替わる「トグルボタン」に変更
  （実際に使うとEscでのキャンセルが多く、ショートカットはあまり使わなかったとのフィードバックから）
- お楽しみコメントの背景色を、バッジと同じ明るい配色に統一
  （真っ黒だとライトモードで浮いて見えるとの指摘から）
- 設定画面(ポップアップ)に「ツールバーにピン留め」のヒントを追加
  （設定の場所が分かりにくいとの指摘。Chromeは拡張機能を既定でパズルピースの中に隠すため）
