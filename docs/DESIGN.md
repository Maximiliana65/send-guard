## 1. 背景・目的

AIチャット（Claude / ChatGPT / Gemini）で入力中に、意図せずEnterキーや送信・再試行ボタンを押してしまい、書きかけのメッセージを送信してしまう事故を防ぎたい。
これまではメモ帳に下書きしてからコピペする運用でしていたが、手間がかかり神経も使うため、Chrome拡張機能として自動化する。

保守しやすさ・拡張しやすさを重視した設計とする。

---

## 2. 全体アーキテクチャ

3層構造にすることで、「1つの機能追加・修正が他に影響しない」設計を目指す。

```
┌─────────────────────────────────────────┐
│ コア（共通ロジック）                        │
│  ・送信ロックガード（Enter/クリック共通判定）  │
│  ・キー入力ハンドラ（改行/送信の判定）         │
│  ・設定ストア（サイト共通の設定管理）          │
└───────────────┬─────────────────────────┘
                │
┌───────────────▼─────────────────────────┐
│ サイトアダプター（接続部分だけ）              │
│  Claude / ChatGPT / Gemini（将来: Grok等）  │
│  → 各サイトの入力欄・送信ボタンの場所を定義   │
└───────────────┬─────────────────────────┘
                │
┌───────────────▼─────────────────────────┐
│ お楽しみプラグイン（後付け・OFF可能）         │
│  絵文字表示・一言コメント・大吉演出 など       │
└─────────────────────────────────────────┘
```

**ポイント**: サイトのUIが変わっても直すのは該当アダプターだけ。新サイト追加もアダプターを1つ足すだけで済む。

## 3. 機能要件

### 3.1 Enterキー・送信の挙動

| 状態 | Enterキー | 送信ボタン・再試行ボタン |
|---|---|---|
| ロック中（通常時） | 改行になるだけ | クリックしても何も起きない |
| ロック解除中 | 送信される | 送信される |

- ロックを解除すると、**次の1回だけ送信可能**。送信した瞬間、自動でまたロックがかかる（毎回慎重に）
- Escキーでロック解除状態をキャンセルできる

### 3.2 ロック解除の方法（両方用意）

1. 画面上の🔒アイコンをクリック
2. キーボードショートカット（例: Ctrl+Shift+U、後で調整可）

### 3.3 対応サイト

- **第一弾**: Claude（claude.ai）のみ実装し、動作確認
- **第二弾**: 動作確認後、ChatGPT・Geminiのアダプターを追加
- 将来的にGrok・Meta AI等を追加する際も、アダプターを1つ追加するだけで対応可能な設計にしておく

### 3.4 お楽しみ機能（将来追加・現時点では設計枠のみ確保）

- 初期状態は **OFF**。設定画面で自分でONにできる
- 発動タイミング: **送信した瞬間**
- 例:「どや！🐧」「おりゃ！🐋」等の一言コメント、🍓💕等の絵文字表示、ごくたまに「大吉🎯」演出
- コア機能（ロック）とは完全に切り離し、後から機能追加・削除・調整してもコアに影響しない設計にする

## 4. 技術方針

- フレームワークは使わず、**素のJavaScript / HTML / CSS**
- Chrome拡張機能の標準規格 **Manifest V3** に準拠
- 設定の保存は Chrome標準の `chrome.storage` を使用（設定ファイルを手動で書き換える必要なし）
- ファイルは機能ごとに分割し、1ファイルが肥大化しないようにする（想定構成は次章）

---

## 5. ファイル構成（想定・実装時に調整あり）

```
extension/
├── manifest.json
├── core/
│   ├── lock-guard.js      … ロックの状態管理・判定
│   ├── key-handler.js     … キー入力の判定（改行/送信）
│   └── settings-store.js  … 設定の保存・読み込み
├── adapters/
│   └── claude.js          … Claude.ai用の入力欄・送信ボタン定義
│                             (第二弾で chatgpt.js / gemini.js を追加)
├── fun/
│   └── comments.js        … お楽しみ機能（後日実装）
├── popup/                 … 設定画面（拡張機能アイコンから開く）
└── icons/
```
